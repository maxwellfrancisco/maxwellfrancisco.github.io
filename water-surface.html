Water Body Surface Area over Time with Python

Tools: ArcPy, Pandas, Raster Algebra.

Overview

This project uses ArcPy to determine the surface area of a water body using the Normalized Difference Water Index (NDWI). The user inputs Landsat imagery, a study-area shapefile, and the name of the water body. The script outputs rasters containing classified water pixels and generates plots and tables using Pandas and Matplotlib to analyze long-term change.

Workflow / Methodology

User Inputs

  • Name of water body (for automatic folder creation and graphing).
  • Landsat NIR/Green bands grouped by year from USGS EarthExplorer.
  • Shapefile defining the study area used for raster clipping.

Processing

  • Identifies NIR and Green bands from file naming conventions.
  • Calculates NDWI.
  • Clips raster outputs to the study area.
  • Reclassifies pixels with NDWI > 0 as water; others as NoData.
  • Calculates surface area in square meters and acres.
  • Loops across yearly imagery.
  • Uses Pandas to generate summary tables and time-series plots.

Limitations

  • Landsat’s 30 m resolution impacts shoreline accuracy.
  • Study areas are derived from TWDB shapefiles with a 100 m buffer.

Possible Improvements

  • Automated image acquisition via the USGS API.
  • Use of NumPy arrays to improve performance.
  • Volume estimation using contours or TINs.

Conclusions

Long-term trends in water surface area were successfully identified.

  • Error relative to TWDB data was consistent per water body.
  • The method allows estimation where historical data is lacking.

Accuracy varied relative to official observations.

  • Underestimation likely due to shoreline misclassification.
  • Higher-resolution datasets (e.g., Sentinel, GEE) may improve accuracy.
